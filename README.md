# DSE17
DSE Project 17: AHAPS

This repository contains the source code for both the preliminary and detailed design tools used throughout this project.

## General Guidelines
A few conventions and guidelines should be followed while working on this project:

1) Avoid working in the exact same file as other people at the same time as them. If you must, consider branching and/or at least working on different functions or classes.
2) Always notify the group and principal maintainer (@Hz2i) of incoming merges.
3) Try not to populate the main directory with a significant amount of wrapper script files, especially if they are meant to be temporary. Whenever possible, branch out and keep main clean.
4) Avoid code that is AI generated, especially in its entirety. You may use AI tools for the purposes of quickly listing function documentation, syntax correction, typo detection, or library function searches, but please try to understand what you are writing and pushing.
5) For major technical questions (especially regarding the basic structure of the repository or its features), it is best to simply approach the principal maintainer.
6) Most objects that have been assumed to be necessary for the detailed design process have already been created. The basic class structures and assumed required imports have been implemented, with comments explaining what should theoretically still be done. DO NOTE: You may assume that these templates are incomplete. If you think additional functions, parameters, or classes are required, you are free to implement them. However, it would be appreciated if the general structure is followed and if the identifiers you choose are clear (it is better to be more verbose, rather than less).
7) Before starting work on any class or main.py file, please take the time to study the examples that are more-or-less complete (look towards the ```materials_components.py``` file). Should any theoretical questions or objections arise, please contact the principal maintainer. Such issues are best solved early, and standardised code is easier to maintain.
